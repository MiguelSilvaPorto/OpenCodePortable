# Plano de Correção: Sincronização de Versão + Correção Groq

## Objetivo
1. Corrigir o desalinhamento entre a versão do binário OpenCode (`1.17.7`) e o plugin `@opencode-ai/plugin` (`1.16.2`)
2. Corrigir o erro LLM do Groq (plugin não consegue autenticar)
3. Adicionar mecanismo automático de sincronização de versão

---

## Problemas Identificados

### Problema 1: Versão do Plugin Desalinhada

| Componente | Versão Atual | Versão Esperada |
|------------|--------------|-----------------|
| OpenCode binary | 1.17.7 | 1.17.7 ✅ |
| `@opencode-ai/plugin` | **1.16.2** ❌ | 1.17.7 |
| `@opencode-ai/sdk` | **1.16.2** ❌ | 1.17.7 |

**Causa:** O `config/package.json` trava a versão em `1.16.2` e não existe mecanismo para atualizar quando o binário muda.

### Problema 2: Groq LLM Erro de Autenticação

| Arquivo | Linha(s) | Problema |
|---------|----------|----------|
| `config/opencode.jsonc` | 3-10 | Falta `apiKeyEnv: "GROQ_API_KEY"` |
| `opencode.bat` | 319-341 | Detecta `GROQ_API_KEY` mas nunca escreve `apiKeyEnv` no JSON |
| `opencode.bat` | 383-405 | Mesmo bug na seção de correção de caminhos |
| `opencode.ps1` | 288 | Escreve `apiKey` (valor bruto) em vez de `apiKeyEnv` (nome da var) |

**Causa Raiz:** O plugin `@renjfk/opencode-voice` espera `apiKeyEnv` (nome da variável de ambiente), mas o código usa `apiKey` (valor da chave). O plugin ignora `apiKey` completamente.

**Fluxo correto do plugin:**
```javascript
// lib/llm-client.js linha ~85
const apiKey = cfg.apiKeyEnv ? process.env[cfg.apiKeyEnv] : null;
```

**Config correto:**
```json
["@renjfk/opencode-voice", {
  "endpoint": "https://api.groq.com/openai/v1",
  "model": "llama-3.3-70b-versatile",
  "apiKeyEnv": "GROQ_API_KEY"
}]
```

## FASE 1: Correção Imediata - Groq (Resolver Agora)

### 1.1 Atualizar `config/opencode.jsonc`
- **Arquivo:** `config/opencode.jsonc`
- **Mudança:** Adicionar `apiKeyEnv`:
  ```jsonc
  "plugin": [
      ["@renjfk/opencode-voice", {
          "endpoint": "https://api.groq.com/openai/v1",
          "model": "llama-3.3-70b-versatile",
          "apiKeyEnv": "GROQ_API_KEY"  // ADICIONAR ESTA LINHA
      }]
  ]
  ```

### 1.2 Corrigir `opencode.bat` - geração de config
- **Arquivo:** `opencode.bat`
- **Local:** Linhas 319-341 (geração de config) e 383-405 (correção de caminhos)
- **Mudança:** Adicionar `apiKeyEnv` quando Groq é detectado:
  ```batch
  if defined GROQ_API_KEY (
      echo     ["@renjfk/opencode-voice", {
      echo       "endpoint": "!VOICE_ENDPOINT!",
      echo       "model": "!VOICE_MODEL!",
      echo       "apiKeyEnv": "GROQ_API_KEY"
      echo     }]
  ) else (
      echo     ["@renjfk/opencode-voice", {
      echo       "endpoint": "!VOICE_ENDPOINT!",
      echo       "model": "!VOICE_MODEL!"
      echo     }]
  )
  ```

### 1.3 Corrigir `opencode.ps1` - geração de config
- **Arquivo:** `opencode.ps1`
- **Locais:** Função `New-DefaultConfig` (linha ~273) e outras 3 ocorrências
- **Mudança:** Substituir `apiKey` por `apiKeyEnv`:
  ```powershell
  # DE:
  $voiceConfig.Add("apiKey", $apiKey)
  # PARA:
  $voiceConfig.Add("apiKeyEnv", "GROQ_API_KEY")
  ```

## FASE 3: Mecanismo de Sincronização Automática

### 3.1 Adicionar `Sync-NpmPlugins` no `opencode.ps1`
- **Arquivo:** `opencode.ps1`
- **Local:** Após função `Download-OpenCodeExe` (linha ~261)
- **Função:** Detecta versão do binário, compara com plugin instalado, executa `npm install` se diferentes
- **Fluxo:**
  1. Obtém versão do binário via `opencode.exe --version`
  2. Lê versão do plugin de `.plugin-version` ou `package-lock.json`
  3. Se diferentes: escreve `package.json` correto e executa `npm install`
  4. Grava marker `.plugin-version` com a versão sincronizada

### 3.2 Adicionar `sync_npm_plugins` no `opencode.bat`
- **Arquivo:** `opencode.bat`
- **Local:** Após sub-rotina `:write_log` (linha ~436)
- **Função:** Equivalente em batch da função PowerShell

### 3.3 Conectar chamadas nos launchers
- **opencode.bat:** Chamar `call :sync_npm_plugins` após verificação de atualização (linha ~120)
- **opencode.ps1:** Chamar `Sync-NpmPlugins` após bloco de atualização (linha ~875)

## Tratamento de Erros

| Cenário | Comportamento |
|---------|---------------|
| Sem internet | Fallback para versão hardcoded, `npm install` falha graciosamente |
| npm não encontrado | Log de warning, continha sem bloquear |
| `npm install` falha | 3 retentativas com backoff, log de erro |
| Binário atualizado | Na próxima inicialização, `Sync-NpmPlugins` detecta e atualiza |
| `node_modules` corrompido | `npm install` é idempotente e repara |
